PRD #2: Refundii (Shopee Direct Engine)

Tài liệu Yêu cầu Hệ thống - Core Logic kết nối API Shopee Open Platform

1. Tổng quan Kiến trúc (Architecture Overview)

  • Mục tiêu: Tối ưu hóa 100% biên lợi nhuận bằng cách kết nối trực tiếp với Shopee Affiliate, loại bỏ mạng lưới trung gian.
  • Nền tảng Tích hợp: Shopee Open Platform (Affiliate API).
  • Phương pháp Tracking: Sử dụng tham số sub_id (truyền ID Tên Ví) trong API tạo link của Shopee để đối soát 1-1.

2. Core Epics & Acceptance Criteria

Epic 1: Product Fetching (Bóc tách dữ liệu sản phẩm)

✅ Tiêu chuẩn nghiệm thu (AC)

  • Sử dụng Web Scraping (cheerio) đọc thẻ og:title và og:image từ link Shopee người dùng dán vào để tối ưu tốc độ, không chờ API Shopee.
  • Nếu bị Shopee chặn bot cào, kích hoạt Fallback: Trả về Tên "Sản phẩm Shopee" và Ảnh mặc định để luồng người dùng không bị gián đoạn.

Epic 2: Shopee Deep Link Generation (Tạo link hoàn tiền)

✅ Tiêu chuẩn nghiệm thu (AC)

  • Backend xây dựng service mã hóa tham số bảo mật (HMAC-SHA256) theo chuẩn tài liệu Shopee Open Platform.
  • Gọi API generate_short_link của Shopee, truyền URL sản phẩm.
  • Tracking: Đẩy wallet_id (Lấy từ profile user) vào trường sub_id trong payload API.
  • Link Universal rút gọn do Shopee trả về (VD: shope.ee/...) sẽ được gắn vào nút CTA mua hàng.

Epic 3: Thuật toán Tạm tính (Expected Cashback)

✅ Tiêu chuẩn nghiệm thu (AC)

  • Sử dụng mức hoa hồng giả định (Average Commission = 3%) nhân với (User Share = 70%) để hiển thị trên UI.
  • Thiết lập chặn trần hiển thị (Cap limit) là 15.000đ. Luôn hiển thị kèm dấu "~" (VD: ~ 12.500đ).

Epic 4: Dual Sync Postback (Đối soát dữ liệu)

✅ Tiêu chuẩn nghiệm thu (AC)

  • Webhook (Push): Mở endpoint /api/webhooks/shopee nhận thông báo đơn hàng từ Shopee. Bóc tách sub_id để cập nhật tiền vào transactions.
  • Polling (Pull) - Fallback: Xây dựng một Cronjob định kỳ (VD: 30 phút/lần) gọi API get_conversion_report của Shopee để đối soát lại những đơn hàng mà Webhook có thể bị rớt do lỗi mạng.
  • Tính toán số tiền chốt: Actual Cashback = Shopee Commission * 70%.

3. Database Schema Updates (Supabase)

Column Name Data Type Description / Logic
id uuid Primary Key
wallet_id uuid Foreign key mapping tới bảng profiles
order_code text (nullable) Mã đơn hàng Shopee trả về qua API
expected_cashback numeric Số tiền dự kiến cào được lúc tạo link
actual_cashback numeric Số tiền thật chốt từ API Conversion
status enum created / pending / approved / rejected

Giai đoạn MVP Mocking: API của Shopee Open Platform cần thời gian để xin cấp quyền. Trong quá trình code hiện tại, BẮT BUỘC sử dụng Mô hình Dữ liệu Giả lập (Mock Service). Viết các hàm giả lập việc sinh mã HMAC, giả lập API gọi đến Shopee và tự động trả về một link shope.ee/mock_link sau 1 giây.
